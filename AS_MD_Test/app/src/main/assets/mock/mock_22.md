# 长文本会话 22：商品详情页改版

商品详情页改版要先明确目标，是提升转化、降低咨询、强化品牌还是承接内容种草。不同目标会影响信息层级。如果当前问题是用户咨询规格和售后较多，那么规格、安装、保修和常见问题应该更靠前；如果问题是转化低，则需要检查价格感知、评价可信度和首屏卖点。

![详情页布局](https://picsum.photos/seed/mock22/520/260)

| 区块 | 当前问题 | 改版建议 | 衡量指标 |
| --- | --- | --- | --- |
| 首屏 | 卖点分散 | 聚焦 3 个核心利益点 | 首屏点击率 |
| 评价 | 缺少筛选 | 增加图片评价和差评解释 | 评价停留 |
| 规格 | 查找困难 | 固定规格入口 | 咨询率 |
| 售后 | 信息靠后 | 前置保修和退换说明 | 退款率 |

详情页不应该堆满模块，而是按用户决策顺序组织内容。用户通常先判断“是不是我要的”，再判断“值不值”，最后判断“买了有没有风险”。改版方案可以参考 [详情页体验规范](https://example.com/docs/pdp-ux)。

```css
.section-title {
  font-weight: 600;
  margin-bottom: 8px;
}
```

上线前建议做 A/B 实验，并分新老用户、渠道来源和商品类目观察。单看整体转化率可能掩盖某些关键人群的体验变化。

补充压测段落：详情页改版建议往往会配合截图、模块表格和样式代码。为了模拟真实 AI 输出，本条数据在图片和表格之后继续追加较长说明。列表快速滑动时，如果这些内容都在 getView 中同步解析，就会放大卡顿；延后到空闲时渲染能让滑动更跟手。

性能观察补充：当 item 高度在渲染完成后变高，ListView 需要保持当前位置稳定，不能因为下方内容撑开就突然跳动到不可预期的位置。
