# 长文本会话 23：营销活动复盘

营销活动复盘不只是看 GMV。需要把目标、预算、人群、渠道、权益、内容和履约都串起来。AI 可以先生成复盘框架，再根据数据填充结论和证据。好的复盘应该明确哪些增长来自活动本身，哪些来自自然波动，哪些是提前透支。

![活动复盘](https://picsum.photos/seed/mock23/520/260)

| 维度 | 观察指标 | 结论示例 | 后续动作 |
| --- | --- | --- | --- |
| 流量 | UV、渠道占比 | 内容渠道拉新明显 | 加大素材复用 |
| 转化 | 下单率、支付率 | 权益门槛偏高 | 调低新客门槛 |
| 客单 | AOV、连带率 | 套装贡献高 | 保留组合推荐 |
| 履约 | 发货时效、投诉 | 高峰期延迟 | 提前排班 |

复盘中要注意归因边界。比如活动期间自然搜索也上升，不能全部算作投放贡献。建议保留对照组或使用历史同期校正。更多模板见 [活动复盘模板](https://example.com/docs/campaign-review)。

```markdown
## 复盘结论
- 达成目标：是
- 最大贡献渠道：内容推荐
- 主要问题：履约延迟
```

复盘最终要沉淀为下一次活动的策略，而不是停留在描述数据。每个结论最好对应一个可执行动作和负责人。

补充压测段落：活动复盘类回复很容易继续展开渠道拆解、预算使用、素材表现和履约复盘。长内容并不一定都在首屏可见，因此初始化时不应把所有 Markdown 都完整构建成 View。更合理的方案是先让消息列表可操作，再把当前屏幕中的消息逐步渲染成完整 Markdown。

性能观察补充：复盘内容常包含多个小标题和列表，渲染完成前后应避免文字重叠、表格抖动和图片区域尺寸突变，保证阅读节奏稳定。
