# 长文本会话 25：门店运营日报

门店运营日报应该让店长快速知道今天哪里好、哪里差、下一步做什么。AI 可以把销售、客流、库存、人员和评价数据合并成摘要。日报不宜太长，首屏要先给结论，然后再提供可展开的数据明细。

![门店日报](https://picsum.photos/seed/mock25/520/260)

| 指标 | 今日 | 环比 | 建议 |
| --- | ---: | ---: | --- |
| 销售额 | 86,300 | +12% | 保持晚高峰导购 |
| 客流 | 2,140 | -3% | 增加门口活动 |
| 连带率 | 1.42 | +0.08 | 推广套装陈列 |
| 缺货 SKU | 18 | +5 | 优先补畅销款 |

日报中的建议要尽量具体。例如不要只说“提升转化”，而要说“晚 18 点到 20 点安排两名导购在新品区”。如果某项指标异常，最好附带可能原因和验证入口。参考 [门店经营指标](https://example.com/docs/store-daily)。

```text
今日重点：晚高峰销售强，但雨具缺货影响连带购买。
建议动作：补货雨具，并在收银台增加组合陈列。
```

门店管理者通常时间有限，所以日报要少讲背景，多给动作。数据明细可以放在二级页，不要挤占结论区。

补充压测段落：门店日报如果覆盖多个门店、多个班次和多个品类，单条回复会自然变长。AI 可能继续说明异常门店、库存短缺、人员排班和顾客评价。对这种内容，列表应该先保证滑动顺畅，再在空闲时完成富文本渲染，让用户能快速定位想看的日期或门店。

性能观察补充：日报类内容适合验证首屏体验，因为用户通常打开页面就开始滚动查找重点。占位和完整渲染之间的切换要足够轻，不应像页面卡死。
