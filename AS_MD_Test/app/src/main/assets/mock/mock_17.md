# 长文本会话 17：库存预测说明

库存预测不能只看最近销量。促销、天气、节假日、渠道库存、供应商交期和替代品都会影响实际需求。AI 在给出补货建议时，应该同时展示预测依据和不确定性区间。对于高价值或长交期商品，宁愿提前暴露风险，也不要只给一个看似精确的数字。

| 商品组 | 当前库存 | 预测销量 | 建议补货 | 置信度 |
| --- | ---: | ---: | ---: | --- |
| 夏季饮品 | 1200 | 2300 | 1500 | 高 |
| 防晒用品 | 430 | 980 | 700 | 中 |
| 雨具 | 260 | 620 | 500 | 中 |
| 礼盒 | 90 | 300 | 260 | 低 |

库存建议要区分“必须补”“建议补”和“观察”。如果供应商交期短，可以降低安全库存；如果交期长或缺货损失高，则提高安全库存。系统还应该展示造成预测变化的因素，例如活动报名、搜索热度提升或区域天气变化。参考 [库存预测口径](https://example.com/docs/inventory-forecast)。

```python
safety_stock = demand_std * service_level_factor * lead_time_days
target_stock = forecast_sales + safety_stock - current_stock
```

预测不是承诺，而是决策辅助。一个好的库存 AI 页面要让采购知道为什么补、补多少、什么时候补，以及如果不补可能发生什么。

补充压测段落：库存预测回复通常会继续解释促销计划、供应商产能、运输周期和安全库存策略。长文本中如果夹杂表格、代码和图片，首次进入页面不应一次性渲染全部历史消息，而应该只处理当前可见内容。这样可以让用户更快进入界面，也能降低低端设备上的主线程压力。

性能观察补充：当用户快速划过多条预测说明时，页面只需要保证列表位置稳定；富文本格式可以在停留后恢复，最终图片、链接和表格仍完整可交互。
