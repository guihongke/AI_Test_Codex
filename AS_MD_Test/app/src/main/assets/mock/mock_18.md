# 长文本会话 18：广告投放诊断

广告投放诊断需要同时看计划、素材、人群、出价和落地页。单纯看到点击率下降，并不能直接判断素材差；也可能是人群疲劳、竞价升高或库存不足导致转化变差。AI 应该先拆解漏斗，再给出优先排查顺序，避免运营同学凭直觉频繁改动。

![投放漏斗](https://picsum.photos/seed/mock18/520/260)

| 漏斗环节 | 指标 | 异常表现 | 建议 |
| --- | --- | --- | --- |
| 展示 | CPM | 成本升高 | 检查竞争和预算节奏 |
| 点击 | CTR | 连续下降 | 更新素材或扩展人群 |
| 转化 | CVR | 波动较大 | 检查落地页和库存 |
| 成本 | CPA | 超出目标 | 调整出价和否定人群 |

诊断结果最好带有“证据”。例如“过去三天 CTR 下降 18%，但 CVR 稳定”，说明素材吸引力可能变弱；如果 CTR 稳定但 CVR 下降，则优先检查落地页、价格和商品状态。策略说明见 [投放诊断手册](https://example.com/docs/ad-diagnosis)。

```javascript
const cpa = spend / conversions;
const ctr = clicks / impressions;
const cvr = conversions / clicks;
```

投放系统最怕频繁小幅调整造成学习期重置。AI 建议应该明确哪些动作可以立即做，哪些动作需要等数据量足够后再判断。

补充压测段落：广告诊断内容往往会附带素材链接、计划链接和多个指标解释。AI 回复越完整，Markdown 渲染压力越大。为了验证真实场景，这里保留较长段落和表格组合，观察进入页面、快速滑动、停止滑动后的渲染节奏是否稳定，避免用户感觉整个页面突然卡住。

性能观察补充：如果后续接入真实投放数据，可以把诊断结果先作为纯数据进入列表，再按可见范围触发 MarkdownView 的完整渲染，减少首屏等待。
