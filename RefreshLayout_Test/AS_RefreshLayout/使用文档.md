## 2. 推荐接入：RecyclerView + XML Progress

### 2.1 XML

```xml
<com.vivo.chat.weight.refresh.smartrefresh.VRefreshRecyclerLayout
    xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/refreshLayout"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    app:srlDragRate="0.5"
    app:srlEnableAutoLoadMore="true"
    app:srlEnableLoadMore="true"
    app:srlHeaderHeight="64dp"
    app:srlFooterHeight="64dp">

    <com.vivo.chat.weight.refresh.smartrefresh.header.VProgressHeader
        android:layout_width="match_parent"
        android:layout_height="64dp">

        <ProgressBar
            android:layout_width="32dp"
            android:layout_height="32dp"
            android:layout_centerInParent="true" />
    </com.vivo.chat.weight.refresh.smartrefresh.header.VProgressHeader>

    <androidx.recyclerview.widget.RecyclerView
        android:id="@+id/recyclerView"
        android:layout_width="match_parent"
        android:layout_height="match_parent" />

    <com.vivo.chat.weight.refresh.smartrefresh.footer.VProgressFooter
        android:layout_width="match_parent"
        android:layout_height="64dp">

        <ProgressBar
            android:layout_width="32dp"
            android:layout_height="32dp"
            android:layout_centerInParent="true" />
    </com.vivo.chat.weight.refresh.smartrefresh.footer.VProgressFooter>

</com.vivo.chat.weight.refresh.smartrefresh.VRefreshRecyclerLayout>
```

直接子 View 推荐严格保持 Header、Content、Footer 顺序，且总数不能超过 3。

### 2.2 Java 初始化

```java
VRefreshRecyclerLayout refreshLayout = findViewById(R.id.refreshLayout);
RecyclerView recyclerView = refreshLayout.getRecyclerView();

recyclerView.setLayoutManager(new LinearLayoutManager(this));
refreshLayout.setAdapter(dataAdapter);

View header = getLayoutInflater().inflate(
        R.layout.view_list_header,
        recyclerView,
        false
);
refreshLayout.addHeaderView(header);

refreshLayout.setOnRefreshListener(layout -> requestFirstPage());
refreshLayout.setOnLoadMoreListener(layout -> requestNextPage());
```

不要改为 `recyclerView.setAdapter(dataAdapter)`，否则列表 Header 不会显示。

### 2.3 完成请求

```java
private void onRefreshResult(boolean success, List<Item> data) {
    if (success) {
        adapter.replaceData(data);
        refreshLayout.resetNoMoreData();
    }
    refreshLayout.finishRefresh(success);
}

private void onLoadMoreResult(boolean success, List<Item> data, boolean noMoreData) {
    if (success) {
        adapter.appendData(data);
    }
    refreshLayout.finishLoadMore(0, success, success && noMoreData);
}
```

`finishLoadMore(boolean)` 的 boolean 表示成功与否，不表示“没有更多数据”。标记最后一页
必须使用三个参数重载或 `finishLoadMoreWithNoMoreData()`。

## 3. 设置滑动阻尼

### 3.1 Java

```java
refreshLayout.setDragRate(0.45f);
```

### 3.2 XML

```xml
app:srlDragRate="0.45"
```

`dragRate = View 实际偏移 / 手指移动距离`。可参考：

| 值 | 体感 | 建议场景 |
| ---: | --- | --- |
| `0.25~0.35` | 阻尼强，需要拖较长距离 | 希望避免误触 |
| `0.4~0.55` | 平衡，接近默认体验 | 普通列表，推荐 |
| `0.6~0.8` | 跟手，较轻 | Header 高度较小 |
| `0.9~1.0` | 阻尼很弱 | 仅建议特殊交互 |

阻尼不会改变 Header 高度。若想改变“拖多少触发”，使用
`setHeaderTriggerRate()` / `setFooterTriggerRate()`；若想限制最大拖动距离，使用
`setHeaderMaxDragRate()` / `setFooterMaxDragRate()`；若想改变松手后的动画速度，使用
`setReboundDuration()`。

```java
refreshLayout
        .setDragRate(0.45f)            // 手指移动到 View 偏移的基础比例
        .setHeaderTriggerRate(1.0f)    // 下拉到 1 个 Header 高度触发刷新
        .setFooterTriggerRate(1.0f)    // 上拉到 1 个 Footer 高度触发加载
        .setHeaderMaxDragRate(2.0f)    // Header 最多拉到自身高度的 2 倍
        .setFooterMaxDragRate(2.0f)
        .setReboundDuration(300);      // 松手和结束回弹的基础时长，毫秒
```

Demo 效果页底部提供阻尼滑杆，可在拖动前实时调整 `0.1~1.0`。

## 4. 切换内置 Header/Footer

### 4.1 Classics

```java
refreshLayout.setRefreshHeader(new VClassicsHeader(this));
refreshLayout.setRefreshFooter(new VClassicsFooter(this));
refreshLayout.setPrimaryColorsId(R.color.refresh_primary, android.R.color.white);
```

`VClassicsHeader` 常用 API：

| API | 说明 |
| --- | --- |
| `setLastUpdateTime(Date)` | 指定上次更新时间 |
| `setTimeFormat(DateFormat)` | 自定义时间格式 |
| `setLastUpdateText(CharSequence)` | 直接指定时间文本 |
| `setEnableLastTime(boolean)` | 是否显示上次更新时间 |
| `setAccentColor(int)` | 设置文字和图标强调色 |
| `setTextSizeTime(...)` | 设置时间文字大小 |
| `setTextTimeMarginTop(...)` | 设置时间文字顶部间距 |

`VClassicsFooter` 支持加载中、完成、失败和没有更多数据状态，文字和图标主要通过 XML
属性配置。

### 4.2 Material

```java
VMaterialHeader header = new VMaterialHeader(this)
        .setShowBezierWave(true)
        .setColorSchemeResources(
                R.color.blue,
                R.color.green,
                R.color.orange
        )
        .setScrollableWhenRefreshing(false);

refreshLayout.setRefreshHeader(header);
```

| API | 说明 |
| --- | --- |
| `setProgressBackgroundColorSchemeResource(int)` | 用资源 ID 设置圆形进度背景色 |
| `setProgressBackgroundColorSchemeColor(int)` | 用 ColorInt 设置进度背景色 |
| `setColorSchemeResources(int...)` | 用颜色资源设置转圈配色 |
| `setColorSchemeColors(int...)` | 用 ColorInt 设置转圈配色 |
| `setSize(int)` | 设置 Material 进度尺寸 |
| `setShowBezierWave(boolean)` | 是否显示贝塞尔波浪背景 |
| `setScrollableWhenRefreshing(boolean)` | 刷新期间 Content 是否可继续滚动 |

### 4.3 任意 View

普通 View 可以包装为刷新组件：

```java
View customHeader = getLayoutInflater().inflate(R.layout.my_header, refreshLayout, false);
refreshLayout.setRefreshHeader(new VRefreshHeaderWrapper(customHeader));

View customFooter = getLayoutInflater().inflate(R.layout.my_footer, refreshLayout, false);
refreshLayout.setRefreshFooter(new VRefreshFooterWrapper(customFooter));
```

Wrapper 适合只需要跟随移动的静态 View。需要按拖动进度改变文字、旋转角度或动画时，
应实现 `VRefreshHeader` / `VRefreshFooter`，或继承 `VSimpleComponent`。

## 5. RecyclerView Header API

| API | 说明 |
| --- | --- |
| `setAdapter(Adapter)` | 设置业务 Adapter，并自动拼接 Header Adapter |
| `getRecyclerView()` | 获取 XML 中的 RecyclerView；找不到时抛异常 |
| `getDataAdapter()` | 获取业务 Adapter，不包含内部 Header Adapter |
| `addHeaderView(View)` | 添加列表 Header；同一实例重复添加会被忽略 |
| `removeHeaderView(View)` | 删除 Header，成功返回 `true` |

可以添加多个 Header：

```java
View notice = inflater.inflate(R.layout.header_notice, recyclerView, false);
View banner = inflater.inflate(R.layout.header_banner, recyclerView, false);
refreshLayout.addHeaderView(notice);
refreshLayout.addHeaderView(banner);
```

在 `GridLayoutManager` 和 `StaggeredGridLayoutManager` 下，Header 会自动占满整行。

## 6. VRefreshLayout 完整业务 API

`VSmartRefreshLayout` 和 `VRefreshRecyclerLayout` 均实现 `VRefreshLayout`。除特殊说明外，
配置方法返回 `VRefreshLayout`，支持链式调用。

### 6.1 尺寸、阻尼和动画

| API | 说明 |
| --- | --- |
| `setHeaderHeight(float dp)` / `setHeaderHeightPx(int)` | 设置 Header 高度 |
| `setFooterHeight(float dp)` / `setFooterHeightPx(int)` | 设置 Footer 高度 |
| `setHeaderInsetStart(float dp)` / `...Px(int)` | Header 起始位置额外偏移 |
| `setFooterInsetStart(float dp)` / `...Px(int)` | Footer 起始位置额外偏移 |
| `setDragRate(float)` | 设置手势阻尼，范围 `0..1`，默认 `0.5` |
| `setHeaderMaxDragRate(float)` | Header 最大拖动倍数，默认 `2.5` |
| `setFooterMaxDragRate(float)` | Footer 最大拖动倍数，默认 `2.5` |
| `setHeaderTriggerRate(float)` | Header 触发阈值倍数，默认 `1.0` |
| `setFooterTriggerRate(float)` | Footer 触发阈值倍数，默认 `1.0` |
| `setReboundDuration(int ms)` | 回弹基础时长，默认 `300ms` |
| `setReboundInterpolator(Interpolator)` | 自定义放手、完成时的回弹曲线 |

### 6.2 组件与内容

| API | 说明 |
| --- | --- |
| `setRefreshHeader(VRefreshHeader)` | 替换 Header |
| `setRefreshHeader(header, width, height)` | 替换 Header 并指定 px 尺寸 |
| `setRefreshFooter(VRefreshFooter)` | 替换 Footer |
| `setRefreshFooter(footer, width, height)` | 替换 Footer 并指定 px 尺寸 |
| `setRefreshContent(View)` | 代码创建页面时设置 Content |
| `setRefreshContent(view, width, height)` | 设置 Content 和 px 尺寸 |
| `getRefreshHeader()` / `getRefreshFooter()` | 获取当前刷新组件，可能为 `null` |
| `getLayout()` | 获取实际 ViewGroup |
| `setPrimaryColors(int...)` | 使用 ColorInt 分发主题色 |
| `setPrimaryColorsId(int...)` | 使用颜色资源 ID 分发主题色 |

### 6.3 功能开关

| API | 说明 |
| --- | --- |
| `setEnableRefresh(boolean)` | 开关下拉刷新，默认开启 |
| `setEnableLoadMore(boolean)` | 开关上拉加载；设置监听器时可智能开启 |
| `setEnableAutoLoadMore(boolean)` | 到达底部后是否自动触发加载，默认开启 |
| `setEnableHeaderTranslationContent(boolean)` | Header 展开时是否移动 Content |
| `setEnableFooterTranslationContent(boolean)` | Footer 展开时是否移动 Content |
| `setEnableOverScrollBounce(boolean)` | 是否启用列表越界回弹 |
| `setEnableOverScrollDrag(boolean)` | 是否允许无刷新动作的越界拖动 |
| `setEnablePureScrollMode(boolean)` | 只保留越界效果，不执行刷新业务 |
| `setEnableNestedScroll(boolean)` | 是否启用嵌套滚动协作 |
| `setEnableLoadMoreWhenContentNotFull(boolean)` | 内容不足一屏时是否允许加载更多 |
| `setEnableScrollContentWhenLoaded(boolean)` | 加载完成时是否调整 Content 显示新数据 |
| `setEnableScrollContentWhenRefreshed(boolean)` | 刷新完成时是否调整 Content |
| `setEnableFooterFollowWhenNoMoreData(boolean)` | 无更多数据时 Footer 是否跟随短内容 |
| `setEnableClipHeaderWhenFixedBehind(boolean)` | FixedBehind Header 是否裁剪 |
| `setEnableClipFooterWhenFixedBehind(boolean)` | FixedBehind Footer 是否裁剪 |
| `setDisableContentWhenRefresh(boolean)` | 刷新中是否禁止操作 Content |
| `setDisableContentWhenLoading(boolean)` | 加载中是否禁止操作 Content |

### 6.4 固定和跟随 View

| API | 说明 |
| --- | --- |
| `setFixedHeaderViewId(int)` | 指定 Content 内固定顶部 View |
| `setFixedFooterViewId(int)` | 指定 Content 内固定底部 View |
| `setHeaderTranslationViewId(int)` | 指定 Header 拖动时跟随位移的 View |
| `setFooterTranslationViewId(int)` | 指定 Footer 拖动时跟随位移的 View |

这些 API 的 ID 必须位于当前 Layout 的 Content 子树中。普通列表一般不需要设置。

### 6.5 监听器

| API | 回调用途 |
| --- | --- |
| `setOnRefreshListener(VOnRefreshListener)` | `onRefresh(layout)` 请求第一页 |
| `setOnLoadMoreListener(VOnLoadMoreListener)` | `onLoadMore(layout)` 请求下一页 |
| `setOnRefreshLoadMoreListener(...)` | 一个对象同时处理刷新和加载 |
| `setOnMultiListener(VOnMultiListener)` | 状态、位移、释放、动画开始和结束的完整观察 |
| `setScrollBoundaryDecider(VScrollBoundaryDecider)` | 自定义何时允许下拉或上拉 |

推荐使用 `VSimpleMultiListener`，只重写需要的回调：

```java
refreshLayout.setOnMultiListener(new VSimpleMultiListener() {
    @Override
    public void onStateChanged(
            VRefreshLayout layout,
            VRefreshState oldState,
            VRefreshState newState
    ) {
        Log.d("Refresh", oldState + " -> " + newState);
    }
});
```

自定义边界示例：

```java
refreshLayout.setScrollBoundaryDecider(new VScrollBoundaryDecider() {
    @Override
    public boolean canRefresh(View content) {
        return !recyclerView.canScrollVertically(-1);
    }

    @Override
    public boolean canLoadMore(View content) {
        return !recyclerView.canScrollVertically(1);
    }
});
```

### 6.6 完成、关闭和无更多数据

| API | 说明 |
| --- | --- |
| `finishRefresh()` | 成功结束刷新 |
| `finishRefresh(int delayed)` | 延迟结束刷新 |
| `finishRefresh(boolean success)` | 按成功/失败结束 |
| `finishRefresh(delayed, success, noMoreData)` | 结束并可同步更新无更多状态 |
| `finishRefreshWithNoMoreData()` | 结束刷新并标记无更多数据 |
| `finishLoadMore()` | 成功结束加载 |
| `finishLoadMore(int delayed)` | 延迟结束加载 |
| `finishLoadMore(boolean success)` | 按成功/失败结束 |
| `finishLoadMore(delayed, success, noMoreData)` | 推荐的分页完成 API |
| `finishLoadMoreWithNoMoreData()` | 结束加载并标记无更多数据 |
| `resetNoMoreData()` | 刷新第一页后恢复可加载状态 |
| `setNoMoreData(boolean)` | 立即设置或取消无更多状态 |
| `closeHeaderOrFooter()` | 强制关闭当前 Header/Footer；不替代正常 finish |

`finishRefresh(..., Boolean noMoreData)` 使用包装类型：传 `null` 表示不修改当前

### 6.7 自动动作和查询

| API | 说明 |
| --- | --- |
| `autoRefresh()` / `autoRefresh(delayed)` | 播放 Header 动画并触发刷新回调 |
| `autoRefreshAnimationOnly()` | 只播放动画，不触发业务回调 |
| `autoRefresh(delayed, duration, dragRate, animationOnly)` | 完整自动刷新参数 |
| `autoLoadMore()` / `autoLoadMore(delayed)` | 播放 Footer 动画并触发加载回调 |

## 8. 全局默认配置

在 `Application.onCreate()` 中设置一次：

```java
VSmartRefreshLayout.setDefaultRefreshInitializer((context, layout) -> layout
        .setDragRate(0.45f)
        .setReboundDuration(300)
        .setEnableLoadMoreWhenContentNotFull(false));

VSmartRefreshLayout.setDefaultRefreshHeaderCreator((context, layout) ->
        new VClassicsHeader(context));

VSmartRefreshLayout.setDefaultRefreshFooterCreator((context, layout) ->
        new VClassicsFooter(context));
```

XML 或 Java 已设置 Header/Footer 时，不会再使用对应默认 Creator。

## 9. 自定义刷新组件

简单组件可以继承 `VSimpleComponent`：

```java
public final class MyHeader extends VSimpleComponent implements VRefreshHeader {

    public MyHeader(Context context) {
        super(context, null, 0);
        mSpinnerStyle = VSpinnerStyle.Translate;
    }

    @Override
    public void onMoving(
            boolean isDragging,
            float percent,
            int offset,
            int height,
            int maxDragHeight
    ) {
        progressView.setRotation(percent * 360F);
    }

    @Override
    public void onStartAnimator(
            VRefreshLayout refreshLayout,
            int height,
            int maxDragHeight
    ) {
        progressView.startAnimation();
    }

    @Override
    public int onFinish(VRefreshLayout refreshLayout, boolean success) {
        progressView.stopAnimation();
        return 0;
    }
}
```

实现 `VRefreshFooter` 时还需实现 `setNoMoreData(boolean)`，返回 true 表示 Footer 能正确
展示无更多数据状态。`VRefreshKernel` 是组件与 Layout 的内部协作接口，普通 Activity、
Fragment、ViewModel 不应直接使用。

## 10. 常见问题

### Header View 不显示

确认使用的是 `refreshLayout.setAdapter(adapter)`，并且 `addHeaderView()` 传入的 View
不是已经固定挂载在其他父布局中的 View。

### 设置监听器后没有加载更多

检查 XML/Java 是否明确调用了 `setEnableLoadMore(false)`。显式关闭属于人工配置，框架
不会因设置监听器再次强制打开；需要调用 `setEnableLoadMore(true)`。

### 刷新一直不结束

网络请求所有成功、失败和异常分支都必须调用 `finishRefresh` 或 `finishLoadMore`。
Activity 销毁后回调应被取消，或先检查生命周期，避免向已销毁页面提交结果。

### 松手后动画太慢或太快

拖动阶段调 `setDragRate`，松手阶段调 `setReboundDuration`。不要通过改变 Header 高度
来模拟动画速度。

### 上拉重复请求

最后一页成功时传 `noMoreData=true`。刷新第一页前或成功后调用 `resetNoMoreData()`，
失败时不要递增页码。
